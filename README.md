# mrkt-resell — твой бот-снайпер подарков на @mrkt

Цель: упростить и сделать прибыльной перепродажу Telegram-подарков на
[@mrkt](https://t.me/mrkt) при маленьком стартовом капитале, с управлением
прямо из твоего Telegram-бота.

Принцип для малых денег: **сначала бесплатно измерь эдж, потом трать деньги на
автоматику покупки.** Проект разбит на этапы.

---

## Что уже готово (Этап 1) — сканер эджа + бот управления

Бот в фоне обходит рынок, считает устойчивый floor по каждой коллекции и
присылает тебе в личку лоты, выставленные ниже floor (потенциальные флипы:
купить → сразу перевыставить у floor). **Ничего не покупает** — только находит
и уведомляет. Задача этапа: за сутки-двое ответить цифрами, есть ли деньги
(сколько флипов в час, какая маржа), прежде чем рисковать капиталом.

Файлы:
- `bot.py` — Telegram-бот (управление из TG). **Главный процесс.**
- `engine.py` — общее ядро (конфиг, поиск флипов, обход рынка).
- `mrkt_client.py` — клиент API, только проверенные эндпоинты `/auth`, `/gifts/saling`.
- `scanner.py` — тот же движок, но CLI (для быстрой проверки с ноутбука + `--probe`).
- `Dockerfile` — окружение для bothost.ru.

Команды бота: `/status`, `/deals`, `/settoken <токен>`, `/watch`, `/unwatch`,
`/margin 12`, `/pause`, `/resume`.

## Что тебе нужно подготовить

1. **Бот у @BotFather** → получи `BOT_TOKEN`.
2. **Свой user_id** у @userinfobot → `ADMIN_ID` (иначе бот слушает всех).
3. **Токен MRKT из DevTools** (бесплатно, ~5 мин):
   - Telegram Web (web.telegram.org) в Chrome → открой @mrkt mini-app.
   - F12 → **Network** → потыкай коллекции → найди запрос к `api.tgmrkt.io`.
   - Скопируй заголовок **`Authorization`** — это `MRKT_TOKEN`.
   - Живёт ~сутки. Обновлять можно прямо в боте: `/settoken <токен>`.
4. **Прокси — НЕ нужен** (хост bothost в Нидерландах, EU-IP `api.tgmrkt.io` не
   режет). `MRKT_PROXY` оставлен как запасной вариант на случай геоблока/лимитов:
   тогда подними SOCKS5 на немецком VPS (ZondVPN) и укажи его. По умолчанию — пусто.

## Запуск локально (проверить, что всё живо)
```bash
cd ~/mrkt-resell
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env          # заполни BOT_TOKEN, ADMIN_ID, MRKT_TOKEN, MRKT_PROXY
set -a; source .env; set +a

python scanner.py --probe     # РАЗОВО: сверь имена полей (см. ниже)
python bot.py                 # запуск бота
```

**Про `--probe` (сделай один раз):** API @mrkt неофициальный, точные имена полей
не задокументированы. `--probe` печатает сырой JSON лота. Если цена/id называются
не так, как в `mrkt_client.py` (`_PRICE_KEYS` / `_ID_KEYS` / `_NUM_KEYS`) —
допиши реальные имена туда. Это единственная ручная подгонка.

## Деплой на bothost.ru

bothost.ru умеет **Docker** и деплой по `git push` — нам подходит.

1. Залей папку в свой Git-репозиторий (`.env`, `config.json`, `deals.jsonl` уже
   в `.gitignore` — секреты не утекут).
2. В bothost создай проект → тип **Custom/Docker** (у нас есть `Dockerfile`),
   подключи репозиторий.
3. В разделе **Environment** задай переменные: `BOT_TOKEN`, `ADMIN_ID`,
   `MRKT_TOKEN`, `MRKT_PROXY` (и при желании `MRKT_COLLECTIONS`, `MRKT_MARGIN`).
4. Deploy. При каждом `git push` bothost пересоберёт и перезапустит бота.

✅ **Про IP.** Хост bothost в Нидерландах — прямой доступ к `api.tgmrkt.io`
работает, прокси не нужен. Если вдруг увидишь 403/ошибки соединения в `/status`
(геоблок или лимиты) — включи `MRKT_PROXY` через немецкий VPS как фолбэк.

⚠️ **Про config.json.** Настройки и токен из `/settoken` пишутся в `config.json`.
Перезапуск он переживает, но **редеплой (git push) может его сбросить** — тогда
бот возьмёт токен снова из env `MRKT_TOKEN`. Держи актуальный токен и там, и там.

## Трезвые ожидания (по фактическим данным маркета, июль 2026)

Провёл разведку через API. Честная картина:

- **Статический флип у floor не работает.** У ликвидных коллекций разброс цен
  у floor всего 0–3%, а комиссия 5%. «Купить самый дешёвый → продать по floor»
  = убыток. Floor эффективно «выбрит» чужими ботами: лоты ниже floor скупаются
  за секунды, поэтому в любом срезе самый дешёвый ≈ floor.
- **Дёшево ≠ выгодно.** Самые дешёвые коллекции (Xmas Stocking и т.п., ~2.46 TON)
  — это floor-спам с нулевым спредом, флипать там нечего.
- **Реальный, но тонкий эдж** — ловить транзиентные недооценки (кто-то по ошибке
  выставил на 8–15% ниже floor) быстрее чужих ботов. На дешёвом хосте выигрывать
  гонку тяжело; ниша — то, что боты проспали (ночь, менее popular коллекции).
- **Глубокий эдж** (если захочешь идти дальше) — оценка справедливой цены по
  комбинации атрибутов (модель+фон+символ) и покупка недооценённых по редкости.
  Требует построить attribute-pricing модель; частично уже заботтлено.
- Прибыль ограничена капиталом: это гринд мелких сумм, не станок.
- Автоторговля может нарушать ToS маркета — риск бана аккаунта реален.

Полезное: эндпоинт `GET /gifts/collections` отдаёт все 126 коллекций с floor и
дневным объёмом — основа для будущего «скаута» коллекций.
